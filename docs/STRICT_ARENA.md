# Строгая арена с валидацией правил

Эта подсистема отслеживает силу бота в играх против разных соперников,
с обязательной проверкой каждого хода по правилам переводного дурака.
Любое нарушение правил приводит к немедленному штрафу — партия
засчитывается как поражение, в лог пишется номер правила и причина.

## Файлы

- `rules_validator.py` — детектор нарушений правил. Plain-data, не зависит
  от C++ движка. Все 9 правил ТЗ покрыты (§1–§9).
- `strict_arena.py` — главный арбитр. Хранит каноническое состояние игры,
  валидирует каждый ход, начисляет штрафы, ведёт Elo.
- `test_strict_arena.py` — юнит-тесты (10 кейсов). Запуск:
  `python training/test_strict_arena.py`.

## Типы штрафов

| Код штрафа          | Когда срабатывает                                          | Winner      |
|---------------------|------------------------------------------------------------|-------------|
| `illegal_move`      | Бот предложил ход, нарушающий §1–§9                        | Противник   |
| `timeout`           | Воркер не ответил за `time_budget + 30s`                   | Противник   |
| `infinite_loop`     | Партия превысила `max_moves` (по умолчанию 600)            | Ничья       |
| `worker_dead`       | Субпроцесс бота упал                                       | Ничья       |

## Использование

### Базовый запуск (один матч, 20 партий)

```bash
# ONNX-модель против pure-MCTS
python training/strict_arena.py \
    --bot-a "onnx:path=checkpoints/current.onnx,device=CPU" \
    --bot-b "pure-mcts:time=0.3,threads=1" \
    --games 20 \
    --time-budget 0.3 \
    --report reports/baseline.json \
    --elo-log reports/elo.jsonl
```

### Сравнение двух версий модели

```bash
python training/strict_arena.py \
    --bot-a "onnx:path=checkpoints/v1.onnx,name=v1" \
    --bot-b "onnx:path=checkpoints/v2.onnx,name=v2" \
    --games 50 \
    --time-budget 0.5 \
    --report reports/v1_vs_v2.json
```

### Против случайного бота (быстрая проверка не нарушает ли модель правила)

```bash
python training/strict_arena.py \
    --bot-a "onnx:path=checkpoints/current.onnx,name=model" \
    --bot-b "random,name=random" \
    --games 30
```

Если `a_penalties > 0` — модель предлагает нелегальные ходы. Смотрите
полный отчёт в JSON — там для каждого нарушения указано правило и ход.

## Формат спецификации бота

```
<kind>:key1=val1,key2=val2,...
```

Где `kind` ∈ {`onnx`, `pure-mcts`, `random`}, а ключи:

| Ключ        | Описание                              | По умолчанию |
|-------------|---------------------------------------|--------------|
| `name`      | Короткое имя для логов                | = вся строка |
| `path`      | Путь к .onnx (только для kind=onnx)   | —            |
| `time`      | Таймаут на ход в секундах             | 0.3          |
| `threads`   | Потоков в ISMCTS                      | 1            |
| `device`    | CPU/CUDA (только для kind=onnx)       | CPU          |
| `env-path`  | Путь к durakk_env.pyd/.so             | .            |

Пример: `onnx:path=m.onnx,device=CUDA,time=0.5,name=alpha_v3`

## Что хранит отчёт

JSON-отчёт содержит:

```json
{

  "bot_b": "pure-mcts",

  "b_wins": 6,

  "b_penalties": 0,
  "elo_a": 1287.3,
  "elo_b": 1112.7,
  "rule_violations": {
    "§6": 1               // ← A один раз подкинул ранг не со стола
  },
  "records": [
    {
      "seed": 1000,
      "winner": "A_penalty",
      "end_reason": "illegal_move",
      "end_rule": "§6",
      "end_ply": 14,
      "moves": [
        {"ply": 0, "bot_name": "alpha_v3", "move": {...}, "valid": true, ...},
        {"ply": 14, "bot_name": "alpha_v3", "move": {...},
         "valid": false, "reason": "подкидывать K♥ нельзя: ранг K ...",
         "rule": "§6", "severity": "critical"}
      ]
    },
    ...
  ]
}
```

## Elo-лог

`--elo-log reports/elo.jsonl` пишет по строке на каждый матч:

```json
{"t": 1719...,"a":"alpha_v3","b":"pure-mcts",
 "elo_a_before":1280,"elo_b_before":1120,
 "elo_a_after":1287,"elo_b_after":1113,
 "score_a":1.0}
```

Для долгосрочного мониторинга силы — стройте график `elo_a_after` по
времени. Если кривая падает — модель регрессирует.

## Покрытые правила

| §  | Правило                                          | Где проверяется                |
|----|--------------------------------------------------|--------------------------------|
| §1 | Колода 36 карт, раздача по 6                     | `_fresh_snapshot`              |
| §3 | Первый ход под козырь наименьшего достоинства    | (упрощённо: всегда Me)         |
| §4 | Атака 1+ картами; защита той же мастью старше    | `validate_attack`, `validate_defend` |
| §4 | Козырь бьёт некозырь; козырь — старшим козырем   | `beats()`                      |
| §5 | Перевод того же ранга                            | `validate_transfer`            |
| §5 | Ограничение перевода по руке соперника           | `validate_transfer`            |
| §5 | Проездной 1 раз за кон                           | (поля есть, генерация — TODO)  |
| §6 | Подкидывание только рангов со стола              | `validate_attack`              |
| §6 | Лимит 6 пар / 5 в первом коне                    | `validate_attack`              |
| §6 | Не больше карт, чем у защитника                  | `validate_attack`              |
| §7 | «Бито» только когда все побито                   | `validate_done`                |
| §7 | «Беру» → вдогонку → забрать → ход у атакующего   | `apply_move_to_snapshot`       |
| §8 | Добор до 6 карт, первым — экс-атакующий          | `_do_refill`                   |
| §9 | Конец игры при пустой колоде и руке              | `_is_game_over`                |

## Архитектура

```
                 ┌────────────────────────┐
                 │   StrictArena (arbiter)│
                 │  ────────────────────  │
                 │  • канонический snap   │
                 │  • validate_move()     │
                 │  • apply_move_to_snap()│
                 │  • Elo-трекинг         │
                 └─────────┬──────────────┘
                           │ ask_action(snap)
              ┌────────────┴────────────┐
              ▼                         ▼
  ┌─────────────────────┐   ┌─────────────────────┐
  │ WorkerProcess (A)   │   │ WorkerProcess (B)   │
  │ • DurakEnv          │   │ • DurakEnv          │
  │ • OnnxNet / null    │   │ • OnnxNet / null    │
  │ • возвращает move   │   │ • возвращает move   │
  └─────────────────────┘   └─────────────────────┘
```

Арена — единственный источник правды. Воркеры только предлагают ходы,
арена их валидирует и применяет. Это гарантирует, что ни один бот не
сможет «обмануть» систему, применив нелегальный ход через свой
внутренний `env.step()`.

## Юнит-тесты

```bash
python training/test_strict_arena.py
# 10 пройдено, 0 провалено.
```

Тесты покрывают:
- Атака → Защита → «бито»
- Перевод со сменой ролей
- «Беру» → pursue → Pass → взятие
- «Бито» требует все побитыми
- Подкидывание ранга не со стола → критический штраф
- Перевод при недостатке карт → критический штраф
- Лимит 5 пар в первом коне
- Козырь бьёт некозырь
- Старший козырь бьёт младшего
- Elo-обновление
