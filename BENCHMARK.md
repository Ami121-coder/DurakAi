# BENCHMARK — замеры силы математического бота

Методика измерения и результаты по этапам усиления (Вариант 4: гибрид
ISMCTS + эндшпиль-minimax + 5 математических усилений).

## Методика

**Инструмент:** `training/arena_math.py` — партии через `durakk_env` (без ONNX),
кандидат vs baseline различаются только `time_budget` на ход. Стороны чередуются
(чётные игры — кандидат за Me, нечётные — за Opp).

**Метрики:**
- `winrate` — доля побед кандидата.
- `elo_diff` — `−400 · log10(1/wr − 1)` (clamped wr ∈ [0.01, 0.99]).
- `sec/партия` — пропускная способность.

**Важные оговорки про дисперсию:**
- Дурак — игра с неполной информацией, дисперсия высока.
- На 10 партиях ±20% winrate — норма. **Для значимых замеров нужно ≥100 партий.**
- При равных time_budget ожидаемо ~50% (санity-чек корректности arena).

**Команды:**
```bash
# Быстрый smoke-test (10 партий, ~1 мин):
python training/arena_math.py --games 10 --cand_time 0.1 --base_time 0.02

# Полный замер (100 партий, ~8 мин):
python training/arena_math.py --games 100 --cand_time 0.3 --base_time 0.05
```

## Заметки

- **Этап 0.1** (bot.cpp): бот теперь по умолчанию работает без нейросети —
  `hasModel=False` подтверждено в sanity test. Это убирает деградацию PUCT на
  uniform priors из RandomNet.
- **Этап 0.2** (bot.cpp): эндшпиль-minimax теперь получает корректное число карт
  соперника (`sampleOppHandForEndgame`) вместо всего `unknownPool`.

  С multi-thread и усилениями должно ускориться.
