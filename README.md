# Помощник: переводной дурак онлайн

Десктопное приложение на **Electron** с C++ движком-советником. Состояние игры
вводится вручную; движок подсказывает оптимальный ход по правилам переводного
дурака.

> **Текущий статус — каркас.** Движок реализует **все правила** игры и отдаёт
> валидный ход на основе эвристики. Профессиональный уровень игры (MCTS /
> expectiminimax) — отдельный этап; структура `bot.cpp` готова к замене.
> Полные правила зафиксированы в [`docs/rules.md`](docs/rules.md).

## Архитектура

```
┌─────────────────────────┐      stdin/stdout (JSON построчно)
│  Electron (renderer)    │      ┌──────────────────────────┐
│  HTML/CSS/JS — ввод     │ ───▶ │  durakk_engine.exe       │
│  состояния, совет бота  │ ◀─── │  (C++17: правила + бот)  │
└─────────────────────────┘      └──────────────────────────┘
        ▲ contextBridge (preload)            ▲ child_process.spawn
        │                                    │
└────────┴────────── main process ───────────┘
                    engine-bridge.js
```

- **C++ движок** — отдельный процесс, общается по stdin/stdout. Не блокирует UI,
  изолирован, собирается чистым CMake + MSVC (нет проблем с ABI Electron).
- **Electron-оболочка** спавнит движок один раз и держит живым всё время работы.

## Установка (один раз)

### 1. Поставить Node-зависимости

```bash
npm install
```

### 2. Поставить C++ компилятор (VS Build Tools 2022)

Нужен для сборки движка. **Потребуется подтверждение UAC** (прав админа в
терминале нет — это нормально).

```bash
winget install Microsoft.VisualStudio.2022.BuildTools --override "--quiet --wait --add Microsoft.VisualStudio.Workload.VCTools --includeRecommended"
```

В состав `Workload.VCTools` входят: MSVC-компилятор и **CMake** (отдельно ставить
не нужно). После установки **перезапустите терминал**, чтобы переменные среды
подхватились.

> Альтернатива — установить через GUI «Visual Studio Installer», отметив
> компонент «Разработка классических приложений на C++» (Desktop development with C++).

## Сборка движка

```bash
npm run build:engine           # configure + build (Release)
npm run build:engine:clean     # с очисткой build/
```

Скрипт сам находит MSVC через `vswhere`, поднимает окружение `vcvars64.bat`,
запускает CMake и кладёт готовый `durakk_engine.exe` в корень проекта и в
`engine/dist/`.

Если exe не собран — приложение всё равно запустится, но в шапке покажет
«движок: не собран», а запросы к боту вернут понятную ошибку.

npm start

## Как пользоваться

1. **Моя рука** — кликните карты, которые у вас на руках (они подсветятся).
2. **Козырь** — выберите масть кнопками ♣♦♥♠.
3. **Колода (ост.)** / **Карт у соперника** — введите числа.
4. **Фаза** (атака/защита), **Чей ход**, **Атакующий** — кто что делает сейчас.
5. **Стол** — нажмите «＋ пара на стол» и заполните атаку/защиту (формат
   `РАНК МАСТЬ`, например `7 clubs`).
6. **💡 Получить совет** — бот вернёт ход и подсветит карту зелёным.
7. **📜 Легальные ходы** — список всех допустимых ходов в текущей позиции.

## Протокол IPC

Одна строка = один JSON. Подробности — в комментариях `engine/src/protocol.h`.

| Команда      | Назначение                                            |
|--------------|-------------------------------------------------------|
| `ping`       | проверка живости                                      |
| `setState`   | передать полное состояние игры                        |
| `decide`     | получить рекомендуемый ход                            |
| `legalMoves` | список всех допустимых ходов (для подсказок в UI)     |
| `validate`   | проверить конкретный ход (ok + причина отказа)        |
